> 在使用Python 装饰器的时候，我们很容易陷入Python 装饰器一个很常见的“认知陷阱”问题：  
> **到底什么时候是“定义”装饰器，什么时候是“使用”装饰器？**  
> 今天一文来给大家讲明白装饰器！

### <span id="_4"></span>一、装饰器是什么？

> ###### <span id="_5"></span>一句话定义:
>
> **“定义是造装饰器；使用是套在函数上那一刻；调用是执行被包装后的函数。”**

在 Python 中，装饰器本质上就是一个“接收函数并返回新函数的函数”。

例如：

``` prism
def decorator(func):
    print("装饰器被定义时执行")
    def wrapper(*args, **kwargs):
        print("函数被调用时执行")
        return func(*args, **kwargs)
    return wrapper
```

当我们写：

``` prism
@decorator
def say_hello():
    print("hello")
```

Python 在加载这段代码时，会**立刻执行 `decorator(say_hello)`**，  
并把返回的新函数（`wrapper`）绑定回原名 `say_hello`。

------------------------------------------------------------------------

### <span id="_vs__35"></span>二、定义 vs 使用

| 阶段 | 名称 | 时机 | 发生了什么 |
|----|----|----|----|
| ① | **定义装饰器** | 定义 `decorator()` 函数时 | 只是创建了一个函数对象，还没用到它 |
| ② | **使用装饰器** | 写 `@decorator` 时 | 解释器会调用 `decorator(被装饰函数)`，返回值替代原函数 |
| ③ | **运行被装饰函数** | 调用 `say_hello()` 时 | 实际调用的是 `wrapper()`，执行装饰逻辑后再执行原函数 |

------------------------------------------------------------------------

### <span id="__45"></span>🔍 三、可打印示例

``` prism
def decorator(func):
    print(">>> 装饰器在使用时执行（定义阶段）")
    def wrapper():
        print(">>> wrapper 在函数运行时执行")
        func()
    return wrapper

print(">>> 准备定义函数...")

@decorator
def hello():
    print("Hello!")

print(">>> 函数定义完成，开始调用...")

hello()
```

输出结果：

    >>> 准备定义函数...
    >>> 装饰器在使用时执行（定义阶段）
    >>> 函数定义完成，开始调用...
    >>> wrapper 在函数运行时执行
    Hello!

解释：

- **第一次打印**：脚本顺序执行；
- **第二次打印**：解释器在遇到 `@decorator` 时调用装饰器；
- **第三次打印**：定义完成；
- **第四次打印**：调用被装饰函数时执行 `wrapper()`。

------------------------------------------------------------------------

### <span id="_85"></span>四、常见误区

1.  **误以为装饰器只在函数调用时执行**  
    实际上，**装饰器在函数定义阶段就会运行一次**。

2.  **误以为 @decorator 是语法糖而非调用**  
    它其实等价于：

``` prism
say_hello = decorator(say_hello)
```

### <span id="_96"></span>五、核心判断逻辑总结

| 判断问题                                 | 答案                           |
|------------------------------------------|--------------------------------|
| 装饰器函数 `def decorator(...)` 出现时？ | **定义装饰器本身**（只是定义） |
| 代码中写 `@decorator` 时？               | **使用装饰器（调用它）**       |
| 被装饰函数调用时？                       | **执行被装饰后的逻辑**         |
