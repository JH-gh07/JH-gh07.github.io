> 还不会使用git?一篇文章承包日常用途！！！  
> “老师再也不用担心我不会使用git了”

### <span id="_6"></span>一、初始设置（首次使用）

#### <span id="1_Git_8"></span>1. 安装和配置Git

``` prism
# 安装Git（如果未安装）
# macOS: brew install git
# Ubuntu: sudo apt install git
# Windows: 下载Git for Windows

# 配置用户信息
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# 查看配置
git config --list
```

#### <span id="2_SSH_23"></span>2. 设置SSH密钥（推荐）

``` prism
# 生成SSH密钥
ssh-keygen -t rsa -b 4096 -C "your.email@example.com"

# 启动ssh-agent
eval "$(ssh-agent -s)"

# 添加SSH密钥到ssh-agent
ssh-add ~/.ssh/id_rsa

# 复制公钥到剪贴板
cat ~/.ssh/id_rsa.pub
# 然后在GitHub Settings > SSH Keys 中添加这个公钥
```

### <span id="GitHub_39"></span>二、创建GitHub仓库

#### <span id="1GitHub_41"></span>方法1：在GitHub网站创建

1.  登录GitHub
2.  点击右上角的 “+” → “New repository”
3.  填写仓库名称、描述
4.  选择公开/私有
5.  可选择添加README、.gitignore、LICENSE
6.  点击"Create repository"

#### <span id="2GitHub_CLI_49"></span>方法2：使用GitHub CLI

``` prism
# 安装GitHub CLI
# macOS: brew install gh
# 其他系统参考官方文档

# 登录
gh auth login

# 创建仓库
gh repo create my-project --public --clone
```

### <span id="_62"></span>三、本地项目初始化

#### <span id="1_64"></span>情况1：新项目

``` prism
# 创建项目目录
mkdir my-project
cd my-project

# 初始化Git仓库
git init

# 创建文件
echo "# My Project" > README.md

# 添加文件到暂存区
git add README.md

# 提交
git commit -m "Initial commit"

# 添加远程仓库
git remote add origin git@github.com:username/my-project.git

# 推送到GitHub
git push -u origin main
```

#### <span id="2_89"></span>情况2：现有项目

``` prism
# 进入项目目录
cd existing-project

# 初始化Git（如果还没有）
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit"

# 添加远程仓库
git remote add origin git@github.com:username/existing-project.git

# 推送
git push -u origin main
```

### <span id="_110"></span>四、日常开发流程

#### <span id="1__112"></span>1. 基本工作流

``` prism
# 1. 查看状态
git status

# 2. 添加修改的文件
git add filename.py          # 添加单个文件
git add .                    # 添加所有修改
git add -A                   # 添加所有修改（包括删除）

# 3. 提交更改
git commit -m "Add new feature"

# 4. 推送到远程仓库
git push origin main
```

#### <span id="2__129"></span>2. 分支开发流程

``` prism
# 创建并切换到新分支
git checkout -b feature/new-feature

# 或者使用新语法
git switch -c feature/new-feature

# 开发完成后提交
git add .
git commit -m "Implement new feature"

# 推送分支到GitHub
git push origin feature/new-feature

# 在GitHub上创建Pull Request
# 合并后删除本地分支
git checkout main
git branch -d feature/new-feature
```

### <span id="Git_150"></span>五、常用Git命令详解

#### <span id="1__152"></span>1. 查看和比较

``` prism
# 查看提交历史
git log
git log --oneline
git log --graph

# 查看文件差异
git diff                     # 工作区vs暂存区
git diff --cached           # 暂存区vs最新提交
git diff HEAD               # 工作区vs最新提交

# 查看远程仓库信息
git remote -v
```

#### <span id="2__168"></span>2. 撤销操作

``` prism
# 撤销工作区修改
git checkout -- filename

# 撤销暂存区修改
git reset HEAD filename

# 撤销最后一次提交（保留修改）
git reset --soft HEAD~1

# 撤销最后一次提交（不保留修改）
git reset --hard HEAD~1
```

#### <span id="3__183"></span>3. 同步操作

``` prism
# 拉取远程更新
git pull origin main

# 获取远程更新但不合并
git fetch origin

# 合并远程分支
git merge origin/main
```

### <span id="_195"></span>六、最佳实践

#### <span id="1__197"></span>1. 提交信息规范

``` prism
# 好的提交信息格式
git commit -m "feat: add user authentication"
git commit -m "fix: resolve login bug"
git commit -m "docs: update README"
git commit -m "refactor: optimize database queries"
```

#### <span id="2_gitignore_206"></span>2. .gitignore文件

``` prism
# 创建.gitignore文件
touch .gitignore

# 常见忽略内容
echo "*.pyc" >> .gitignore
echo "__pycache__/" >> .gitignore
echo ".env" >> .gitignore
echo "node_modules/" >> .gitignore
echo ".DS_Store" >> .gitignore
```

#### <span id="3__219"></span>3. 分支策略

``` prism
# 主要分支
main/master     # 生产分支
develop         # 开发分支

# 功能分支
feature/xxx     # 新功能
bugfix/xxx      # 修复bug
hotfix/xxx      # 紧急修复
```

### <span id="_231"></span>七、常见问题解决

#### <span id="1__233"></span>1. 推送被拒绝

``` prism
# 原因：远程有新提交
# 解决：先拉取再推送
git pull origin main
git push origin main

# 或者使用rebase
git pull --rebase origin main
git push origin main
```

#### <span id="2__245"></span>2. 合并冲突

``` prism
# 手动解决冲突后
git add conflicted-file
git commit -m "Resolve merge conflict"
git push origin main
```

#### <span id="3__253"></span>3. 修改最后一次提交

``` prism
# 修改提交信息
git commit --amend -m "New commit message"

# 添加文件到最后一次提交
git add forgotten-file
git commit --amend --no-edit
```

### <span id="_263"></span>八、完整示例流程

``` prism
# 1. 克隆仓库（如果是协作项目）
git clone git@github.com:username/project.git
cd project

# 2. 创建功能分支
git checkout -b feature/user-login

# 3. 开发功能
# ... 编写代码 ...

# 4. 提交更改
git add .
git commit -m "feat: implement user login functionality"

# 5. 推送分支
git push origin feature/user-login

# 6. 在GitHub创建Pull Request

# 7. 合并后清理
git checkout main
git pull origin main
git branch -d feature/user-login
```

这个流程涵盖了从初始设置到日常开发的完整Git和GitHub使用逻辑。掌握这些步骤后，你就能高效地管理代码版本和协作开发了。
